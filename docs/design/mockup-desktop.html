Design spec — for the Slint port

Everything below is the contract between this mockup and .slint. Tokens are named exactly as they should appear in a Slint global singleton.

Categorical topic palette — CVD-validated

topic-1 rust#C8822A · #C07C15
topic-2 local#17A183 · #008A66
topic-3 ai#8672E8 · #6D55D9
topic-4 wasm#C75F81 · #C24D75
topic-5 steel#4C82C4 · #3D6FA8

Hex pairs are dark-surface · light-surface variants. Both sets pass all six palette checks (OKLCH lightness band, chroma floor, protan/deutan/tritan adjacent-pair separation, normal-vision floor, 3:1 contrast vs surface). Hues are assigned to topics in fixed order and never cycled — a 6th topic folds into steel or gets small-multiple treatment.

Ground & accent

ground#101419 · #EDF0F4
surface#1B212B · #FFFFFF
raised#222A35 · #F6F8FB
pulse accent#EFA43F · #B97C1E
ink#E9EDF3 · #1A2230

Neutrals carry a deliberate blue bias (the instrument chassis); the lone warm element is the amber pulse. Amber is spent in exactly three places — waveform, active tool calls, budget meter — so it always means "the pulse is speaking."

Type

  • Instrument Sans 400–700 — all UI text. Titles 620–680 weight, −0.01em tracking.
  • JetBrains Mono 400–700 — every number, label, readout, tool call, and channel name. Always tabular-nums. Eyebrows: 9.5–11px, +0.12em tracking, uppercase.
  • Rule: text wears ink tokens, never a topic color — the colored square beside a label carries identity.

Slint port notes

  • Layout: 262 / flex / 344 three-column HorizontalLayout; spacing scale 4·8·12·16·24; radii 10 (chips/channels), 12 (cards), 14 (window).
  • Sparklines: draw with Path from the same point arrays — 2px stroke, 12% area fill, 3px endpoint dot, dashed 7-day baseline in the large variant.
  • Waveform: static Path + animated dash sweep (animate on dash offset); disable under reduced-motion.
  • Fader: TouchArea + Rust-side hit test (patchboard technique); mute is a latching toggle.
  • Ship both woff2 fonts with the binary; Slint loads them at startup — no system-font fallback drift.
  • Fallback: this HTML is the webapp Plan B if Slint runs out of runway — same tokens, same layout.
